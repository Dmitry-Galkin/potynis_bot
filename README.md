# Potyanis Bot
Telegram-бот для записи на занятия йогой: пользователи могут записываться и отменять запись, админы управляют расписанием и смотрят записи по датам.
## Возможности
- Запись на занятие через `/join` с подтверждением.
- Отмена записи через `/leave` с подтверждением.
- Просмотр расписания (`/schedule`), личных записей (`/my_bookings`) и общей записи (`/all_bookings`).
- Разделение прав доступа:
  - `ADMIN`: доступ к админ-панели `/admin`.
  - `USER`: участник группы, может пользоваться пользовательскими командами.
  - `GUEST`: не участник группы, получает подсказку вступить в группу.
- Админ-панель:
  - добавить шаблон занятия;
  - редактировать шаблон занятия;
  - деактивировать шаблон занятия;
  - посмотреть записи на конкретную дату;
  - добавить отпуск;
  - удалить отпуск.
## Стек
- Python 3.14
- aiogram 3
- SQLite (через `aiosqlite`)
- pandas (для табличной обработки и форматирования данных)
## Требования
- Python 3.14+
- Telegram Bot Token
- Telegram ID группы, в которой работают пользователи бота
- Список Telegram ID админов
## Конфигурация
### 1) Переменные окружения (`.env`)
Создайте файл `.env` в корне проекта:
```env
BOT_TOKEN=your_telegram_bot_token
GROUP_ID=-1001234567890
ADMIN_IDS=111111111,222222222
```
Где:
- `BOT_TOKEN` — токен бота от BotFather.
- `GROUP_ID` — id Telegram-группы (можно получить через команду `/get_chat_id`).
- `ADMIN_IDS` — список id админов через запятую.
### 2) YAML-конфиг
По умолчанию используется `config.yaml`:
- `db.path` — путь к SQLite-базе (например, `data/yoga_classes`)
- `booking.window_days` — горизонт генерации занятий на N дней вперед
- `time.local_timezone` — локальная таймзона для отображения дат/времени
Для разработки есть `config_dev.yaml`.
## Установка и запуск (локально)
```bash
python -m venv .venv
# Windows PowerShell
.venv\Scripts\Activate.ps1
# Linux/macOS
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```
При старте бот:
- выставляет команды Telegram;
- инициализирует таблицы в SQLite (если их нет);
- запускает polling.
## Запуск в Docker
```bash
docker build -t potyanis-bot .
docker run --rm \
  --env-file .env \
  -v %cd%/data:/potyanis_bot/data \
  potyanis-bot
```
Для Linux/macOS замените `%cd%` на `$(pwd)`.
## Команды бота
### Для пользователей
- `/join` — записаться на занятие
- `/leave` — отменить запись
- `/schedule` — посмотреть расписание
- `/my_bookings` — посмотреть свои записи
- `/all_bookings` — посмотреть общую запись
- `/help` — справка
- `/cancel` — отмена текущего действия
### Для админов
- `/admin` — открыть админ-панель
- `/get_chat_id` — вывести id текущего чата (служебная команда)
## Структура проекта
```text
app/
  bot/
    filters/        # Фильтры ролей (admin/user/guest)
    handlers/       # Обработчики пользовательских и админских сценариев
    keyboards/      # Inline-клавиатуры
    middlewares/    # Middleware (логирование, проброс config/bot)
    roles/          # Логика определения роли пользователя
    utils/          # Вспомогательные функции
    interface/      # Регистрация bot-команд
  config/           # Загрузка и схемы конфигурации
  db/               # Слой работы с SQLite и схема таблиц
main.py             # Точка входа
config.yaml         # Основной runtime-конфиг
config_dev.yaml     # Конфиг для разработки
```
## Примечания
- База создается автоматически при первом запуске.
- Пользователь считается `USER`, если состоит в указанной группе.
- В сообщениях используется локальная таймзона из `config.yaml`.
